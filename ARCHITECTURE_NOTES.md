# Architecture Notes

## Source Project Analysis

The original project (`mapmaker/vancouver-map`) was a single-file static HTML/CSS/JS app:
- No build system, no package manager, no framework
- MapLibre GL JS loaded from CDN for interactive map rendering
- CARTO Voyager raster basemap tiles
- Location data stored in a static `locations.json` file
- Photos (JPEG) and audio (MP3) served as local static assets
- Slide-out panel UI with photo gallery, audio player, and text content
- Mobile-responsive with bottom sheet on small screens
- No database, no server, no auth

## New Project Architecture

### Stack
- **Build**: Vite + TypeScript (vanilla, no framework)
- **Map**: MapLibre GL JS (npm package, not CDN)
- **Database**: Supabase (PostgreSQL) for location data
- **Styling**: Vanilla CSS (same design, extracted to separate file)
- **Assets**: Static photos/audio in `public/` directory

### Key Decisions
1. **Vite over plain HTML**: Enables TypeScript, CSS imports, env var handling, and proper bundling
2. **Supabase for data**: Locations now fetched from Supabase with a static fallback for offline/error resilience
3. **Fallback pattern**: If Supabase is unreachable, the app uses hardcoded fallback data so it never breaks
4. **RLS enabled**: `locations` table has row-level security with a public read policy (no write from client)
5. **Migrations as code**: Schema defined in SQL migrations, seed data in `supabase/seed.sql`
6. **Type safety**: Database types defined in `src/types/database.ts`, app code is fully typed

### Directory Layout
```
selfcare-map/
  index.html              # entry point
  src/
    main.ts               # app logic (map, panel, gallery, audio)
    style.css             # all styles
    supabase.ts           # supabase client init
    locations.ts          # data fetching with fallback
    types/
      database.ts         # manual DB types
      database.generated.ts  # (generated by supabase CLI)
    vite-env.d.ts         # env type declarations
  public/
    img/                  # location photos
    audio/                # location audio clips
  supabase/
    config.toml           # supabase project config
    migrations/           # SQL migrations
    seed.sql              # seed data
```

### Data Flow
1. App loads -> MapLibre GL initializes with CARTO basemap
2. `fetchLocations()` queries Supabase `locations` table
3. If Supabase fails, falls back to hardcoded data
4. Markers placed on map for each location
5. Click marker -> panel slides in with photos, text, audio
6. Gallery supports multiple photos with crossfade transitions
7. Audio auto-plays on location select (with browser autoplay priming)
